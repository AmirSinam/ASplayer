# چطور ASplayer را به یک اپلیکیشن تبدیل کنم؟

خروجی نهایی یک فایل `.apk` است. همان فایل را در تلگرام می‌گذاری و هر کس رویش بزند
روی گوشی اندرویدش نصب می‌شود.

---

## وضعیت فعلی سیستم تو

- **فلاتر ۳.۴۴.۴** در `C:\src\flutter` نصب شد. ✅
- **بسته‌ها** گرفته شدند و **کد بدون خطا تحلیل شد**. ✅
- **اندروید اس‌دی‌کی** نصب **نیست** و سرور دانلودش (`dl.google.com`) از این شبکه بسته است. ❌

پس **بیلد محلی فعلاً ممکن نیست** و باید از گیت‌هاب استفاده کنی (راه اول).

### فلاتر روی این شبکه

سرورهای اصلی فلاتر بسته‌اند. قبل از هر دستور فلاتر، این سه خط را در پاورشل بزن:

```powershell
$env:PUB_HOSTED_URL = "https://pub.flutter-io.cn"
$env:FLUTTER_STORAGE_BASE_URL = "https://storage.flutter-io.cn"
$env:Path = "C:\src\flutter\bin;" + $env:Path
```

بدون این‌ها هر دستور فلاتر با خطای شبکه شکست می‌خورد.

---

دو راه هست. **راه اول تنها راهی است که همین حالا کار می‌کند.**

---

## راه اول — بدون نصب هیچ چیز، روی گیت‌هاب

گیت‌هاب برایت بیلد می‌گیرد. نه فلاتر لازم داری، نه اندروید استودیو.

### ۱. حساب گیت‌هاب بساز (اگر نداری)
`github.com` — رایگان.

### ۲. کد را بفرست بالا

در پوشه‌ی `asplayer`:

```bash
git init
git add .
git commit -m "ASplayer، نسخه‌ی اول"
git branch -M main
git remote add origin https://github.com/<نام-کاربری>/asplayer.git
git push -u origin main
```

مخزن را **عمومی** بگذار تا دقیقه‌های بیلد رایگان و نامحدود باشند.

### ۳. منتظر بمان

با همان `push`، بیلد خودکار شروع می‌شود. در صفحه‌ی مخزن، تب **Actions**:

- **سبز شد؟** پایین صفحه بخش **Artifacts** → فایل `asplayer-apk` را دانلود کن.
  از حالت زیپ خارجش کن، داخلش `app-release.apk` است. کارت تمام است.
- **قرمز شد؟** روی مرحله‌ی قرمز بزن، متن خطا را کپی کن و برایم بفرست.

**انتظار داشته باش بار اول قرمز شود.** این کد هیچ‌وقت اجرا نشده و نسخه‌ی بسته‌ها
تست نشده است. یکی دو رفت‌وبرگشت طبیعی است.

---

## راه دوم — روی کامپیوتر خودت

این راه کندتر شروع می‌شود ولی از آن به بعد همه چیز سریع است: تغییر کد، دیدن نتیجه
روی گوشی، رفع خطا. برای ادامه‌ی پروژه لازمش داری.

**ولی یک مانع دارد:** اندروید اس‌دی‌کی از `dl.google.com` دانلود می‌شود و آن سرور
از این شبکه بسته است. برای این یک بار (حدود ۳ تا ۵ گیگابایت) به وی‌پی‌ان نیاز داری.
بعد از نصب اس‌دی‌کی، دیگر لازمش نداری.

### ۱. فلاتر — ✅ انجام شد
در `C:\src\flutter` نصب است. اگر می‌خواهی در هر ترمینال جدید در دسترس باشد،
مسیر `C:\src\flutter\bin` را به `PATH` ویندوز اضافه کن.

### ۲. اندروید اس‌دی‌کی
اندروید استودیو را باز کن. خودش می‌پرسد که اس‌دی‌کی را نصب کند. **وی‌پی‌ان را روشن نگه دار**
تا دانلود تمام شود.

### ۳. سلامت نصب را بررسی کن

```powershell
flutter doctor
flutter doctor --android-licenses
```

### ۴. اسکلت پروژه — ✅ انجام شد
اگر روزی دوباره `flutter create` زدی، حتماً بعدش این را بزن:

```powershell
powershell -ExecutionPolicy Bypass -File android_overrides\apply.ps1
```

این دو فایل اندرویدی را سر جایشان می‌گذارد. بدون آن، پخش در پس‌زمینه و
«اشتراک‌گذاری از تلگرام» کار نمی‌کنند.

### ۵. روی گوشی اجرا کن

روی گوشی اندروید: تنظیمات → درباره‌ی گوشی → هفت بار روی «شماره‌ی ساخت» بزن تا
حالت توسعه‌دهنده روشن شود. بعد در گزینه‌های توسعه‌دهنده، **اشکال‌زدایی یو‌اس‌بی** را روشن کن.

گوشی را با کابل وصل کن و بزن:

```bash
flutter devices
flutter run
```

### ۶. فایل نصبی بساز

```bash
flutter build apk --release
```

خروجی:
`build\app\outputs\flutter-apk\app-release.apk`

## پخش فایل

فایل `app-release.apk` را در تلگرام بگذار. هر کس رویش بزند:

۱. اندروید می‌پرسد «نصب از این منبع مجاز است؟» → اجازه بدهد.
۲. اپ نصب می‌شود.

بدون انقضا، بدون اکانت، بدون هزینه.

### چند نکته
- **امضا:** فایل با کلید اشکال‌زدایی امضا می‌شود. برای پخش در تلگرام مشکلی ندارد،
  ولی برای گوگل‌پلی باید کلید انتشار بسازی. آن وقتش که رسید بگو.
- **حجم:** حدود ۲۰ تا ۳۰ مگابایت. با `--split-per-abi` می‌شود کوچک‌ترش کرد.
- **به‌روزرسانی:** نسخه‌ی جدید را باید دستی بفرستی. اگر شماره‌ی نسخه در `pubspec.yaml`
  را بالا نبری، اندروید نصب روی نسخه‌ی قبلی را قبول نمی‌کند.
