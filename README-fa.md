# ASplayer — نسخه‌ی فلاتر

یک کد، دو پلتفرم. فعلاً هدف اندروید است؛ آی‌فون بعداً از همین کد بیلد می‌شود.

## وضعیت
کد نوشته شده، **هیچ‌وقت اجرا نشده**. اولین `flutter run` احتمالاً چند خطا می‌دهد.
مهم‌ترین ریسک، نسخه‌ی بسته‌هاست: من نتوانستم نسخه‌های `pubspec.yaml` را تست کنم.

## ساخت اپلیکیشن
مراحل کامل، از صفر، در `BUILD-fa.md` است. دو راه دارد: بیلد روی گیت‌هاب بدون نصب چیزی،
یا نصب فلاتر روی ویندوز و اجرا روی گوشی خودت.

**نکته‌ی مهم:** بعد از هر `flutter create`، حتماً `android_overrides\apply.ps1` را بزن.
وگرنه پخش در پس‌زمینه و «اشتراک‌گذاری از تلگرام» کار نمی‌کنند.

## اضافه کردن آهنگ
- **از تلگرام:** فایل صوتی → اشتراک‌گذاری → ASplayer
- **از مدیر فایل:** روی فایل بزن → «باز کردن با» → ASplayer
- **از داخل اپ:** دکمه‌ی + بالای صفحه‌ی خانه

فایل داخل حافظه‌ی خصوصی اپ کپی می‌شود و دیگر به منبع اصلی نیازی نیست.

## ساختار

| مسیر | کار |
|---|---|
| `lib/main.dart` | راه‌اندازی سرویس صدا و پرووایدرها |
| `lib/app_state.dart` | زبان، تم، تب انتخاب‌شده |
| `lib/l10n.dart` | ترجمه‌ها (انگلیسی پیش‌فرض، فارسی اختیاری) |
| `lib/theme.dart` | رنگ‌ها و ویجت شیشه |
| `lib/models.dart` | آهنگ، لیست پخش، دسته‌بندی، مرتب‌سازی |
| `lib/data/library_store.dart` | نگهداری کتابخانه در یک فایل جیسون |
| `lib/data/importer.dart` | کپی فایل و خواندن تگ‌ها |
| `lib/audio/player_controller.dart` | پخش، صف، تصادفی، تکرار، تایمر خواب |
| `lib/screens/` | خوش‌آمد، خانه، مورد علاقه، کتابخانه، پخش، صف، تنظیمات |

## تفاوت‌ها با نسخه‌ی سوییفت
- **لغزنده‌ی صدا** صدای خود اپ را عوض می‌کند، نه صدای کل گوشی.
- کتابخانه به‌جای پایگاه داده در یک فایل جیسون ذخیره می‌شود. برای چند صد آهنگ کافی است؛
  اگر کتابخانه خیلی بزرگ شد باید به `sqflite` مهاجرت کنیم.
