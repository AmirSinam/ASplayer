# انتشار ASplayer

## کلید امضا

کلید انتشار ساخته شده و **بیرون از این مخزن** است:

```
C:\Users\Amirsina\keys\asplayer-release.jks          ← خود کلید
C:\Users\Amirsina\keys\asplayer-key-password.txt     ← رمز
C:\Users\Amirsina\keys\asplayer-keystore-base64.txt  ← همان کلید، برای گیت‌هاب
```

اعتبارش تا حدود سال ۱۴۳۲ است. نام مستعار کلید: `asplayer`

### این را جدی بگیر

اگر این فایل یا رمزش را گم کنی:

- دیگر **هیچ‌وقت** نمی‌توانی نسخه‌ی جدید ASplayer را منتشر کنی.
- کاربران فعلی نمی‌توانند به‌روزرسانی کنند. باید اپ را حذف کنند و دوباره نصب کنند،
  و **همه‌ی آهنگ‌ها و لیست‌های پخششان پاک می‌شود.**
- در گوگل‌پلی، اپ عملاً می‌میرد و باید با شناسه‌ی جدید از صفر شروع کنی.

نه من، نه گوگل، نه هیچ‌کس دیگر نمی‌تواند بازیابی‌اش کند. **از هر سه فایل پشتیبان بگیر**
و در جایی امن نگه‌شان دار — نه در همین کامپیوتر، نه در مخزن گیت.

هرگز این فایل‌ها را کامیت نکن. `.gitignore` جلویشان را می‌گیرد، ولی حواست باشد.

## سکرت‌های گیت‌هاب

بدون این‌ها، بیلد گیت‌هاب با کلید اشکال‌زدایی امضا می‌کند و در لاگ هشدار می‌دهد.
آن خروجی را **منتشر نکن**.

در صفحه‌ی مخزن: **Settings → Secrets and variables → Actions → New repository secret**

چهار سکرت بساز:

| نام | مقدار |
|---|---|
| `ANDROID_KEYSTORE_BASE64` | کل محتوای `asplayer-keystore-base64.txt` |
| `ANDROID_STORE_PASSWORD` | محتوای `asplayer-key-password.txt` |
| `ANDROID_KEY_PASSWORD` | همان رمز (یکی هستند) |
| `ANDROID_KEY_ALIAS` | `asplayer` |

فایل base64 یک خط بلند است (حدود ۵۲۰۰ کاراکتر). همه‌اش را کپی کن، بدون خط جدید.

بعد از تنظیم سکرت‌ها، بیلد بعدی هم فایل نصبی امضاشده می‌سازد و هم بسته‌ی گوگل‌پلی.

## بیلد محلی

فایل `android/key.properties` از قبل ساخته شده و به کلید اشاره می‌کند. کامیت نمی‌شود.

```powershell
$env:PUB_HOSTED_URL = "https://pub.flutter-io.cn"
$env:FLUTTER_STORAGE_BASE_URL = "https://storage.flutter-io.cn"
$env:Path = "C:\src\flutter\bin;" + $env:Path

flutter build apk --release        # برای تلگرام
flutter build appbundle --release  # برای گوگل‌پلی
```

(هنوز اندروید اس‌دی‌کی نصب نیست، پس این‌ها فعلاً کار نمی‌کنند.)

## هر بار که نسخه‌ی جدید می‌دهی

در `pubspec.yaml`، خط `version` را بالا ببر:

```yaml
version: 0.1.0+1
#        ^^^^^ نسخه‌ای که کاربر می‌بیند
#              ^ شماره‌ی بیلد — این باید همیشه بیشتر شود
```

اگر شماره‌ی بیلد را بالا نبری، اندروید نصب روی نسخه‌ی قبلی را قبول نمی‌کند
و گوگل‌پلی هم بسته را رد می‌کند.

عدد نسخه در صفحه‌ی «درباره» هم نشان داده می‌شود؛ آن را در `lib/l10n.dart`
(`appVersion`) به‌روز کن.

## چیزهایی که هنوز برای انتشار جدی کم است

- **آیکن اپ.** فعلاً آیکن پیش‌فرض فلاتر است.
- **فشرده‌سازی کد (R8).** عمداً خاموش است: احتمال دارد کلاس‌های پخش‌کننده را حذف کند
  و تا وقتی اپ روی گوشی واقعی تست نشده، روشن کردنش ریسک است.
- **صفحه‌ی حریم خصوصی.** برای گوگل‌پلی اجباری است. اپ هیچ داده‌ای جمع نمی‌کند،
  ولی باید همین را جایی بنویسی.
- **تست روی گوشی واقعی.** هنوز حتی یک بار اجرا نشده.
